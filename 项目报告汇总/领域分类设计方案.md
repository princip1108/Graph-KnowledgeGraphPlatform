# 知识图谱平台全领域分类体系设计方案 (Expanded V2)

为了构建一个“包罗万象”的知识图谱平台，我们需要一套极为丰富且颗粒度精细的分类体系。本方案在原有基础上进行了大幅扩展，旨在通过多维度、多层级的分类来满足各类垂直领域和长尾场景的需求。

---

## 1. 自然科学与技术 (STEM & Research)

### 🔬 基础科学
*   **物理与天文**: 粒子物理、天体运行、力学结构、量子力学
*   **化学与材料**: 元素周期、分子结构、高分子材料、化学反应
*   **生物科学**: 细胞结构、遗传基因、物种分类、生态系统
*   **地球科学**: 地质构造、气象气候、海洋地理、环境监测

### 💻 计算机与信息技术
*   **人工智能 (AI)**: 神经网络、机器学习算法、LLM知识库
*   **软件工程**: 代码依赖图、微服务架构、API调用链、设计模式
*   **网络安全**: 攻击路径、漏洞关联、威胁情报 (CTI)、APT组织
*   **数据与算法**: 数据库Schema、算法逻辑、数据血缘

### ⚙️ 工程与制造
*   **工业4.0**: 生产产线、数字孪生、设备维护 (IoT)
*   **航空航天**: 飞行器系统、轨道参数、航天工程
*   **建筑与土木**: BIM模型关联、城市管网、交通规划
*   **能源电力**: 智能电网、能源转换、石油化工流程

## 数据库设计扩展建议

为了支持如此丰富的层级结构，建议在该方案下，`Categories` 表增加 `parent_id` 字段以支持无限层级分类：

```sql
CREATE TABLE categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    code VARCHAR(50) NOT NULL,
    parent_id INT DEFAULT NULL, -- 一级分类的 parent_id 为 NULL
    icon VARCHAR(50),
    sort_order INT DEFAULT 0,
    FOREIGN KEY (parent_id) REFERENCES categories(id)
);
```

这样用户在选择时，可以先选大类（如“科学技术”），再联动选择子类（如“人工智能”），甚至三级分类。
